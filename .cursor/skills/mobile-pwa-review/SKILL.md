# Mobile PWA Review

## Checklist

- iOS and Android PWA safe-area behavior is preserved.
- Keyboard behavior does not hide important controls or cause disruptive jumps.
- Dynamic viewport height works on mobile browsers and installed PWAs.
- Bottom overlap is checked for fixed controls, sheets, and maps.
- Horizontal overflow is avoided.
- Map containers size and layer correctly.
- Address suggestions and autocomplete overlays do not jump, clip, or overflow.
- Compact mobile card layout stays readable and usable.
